# 一个常见的管理系统是由那些部分构成的

# 数据库

# 结构化数据库

结构化数据库就是一个特别大型的有特定的查看方式的 Excel 表，这个表中每一列是什么样的都要求是提前确定好的，有固定的结构和数值的类型。

这些数据库通常使用 SQL（Structured Query Language）进行数据查询和操作。

结构化数据库适用于需要强调数据一致性、事务处理和复杂查询的应用场景，如企业级应用、电子商务等。

# 非结构化数据库

非结构化数据库是用于存储无固定格式或结构的数据，如文本、图像、音频、视频等。

它们不使用传统的表格结构，而是采用其他方式来组织和检索数据。常见的非结构化数据库包括：

  • 文档数据库：如 MongoDB，用于存储和查询文档格式数据。
  • 键值存储数据库：如 Redis，用于高速缓存和键值对存储。

非结构化数据库适用于需要存储大量文本、多媒体等非结构化数据的场景，如社交媒体、日志分析等。

# 图数据库

图数据库是一种专门用于存储和查询图结构数据的数据库类型。图数据库以节点和边的形式表示数据，适用于处理实体之间复杂的关系和连接。常见的图数据库包括：

  • Neo4j：一种流行的图数据库，用于处理复杂的关系网络数据。

图数据库适用于需要强调数据之间关系、社交网络分析、知识图谱等场景。

# 向量型数据库

向量型数据库是一种专门用于存储和处理向量数据的数据库类型。在人工智能和机器学习领域，数据通常以向量的形式表示，例如图像特征、文本嵌入等。

可以实现类似图搜图、文搜文这种查找相似的事务的功能。

# 后台

系统架构设计包括各个模块之间的组织方式和交互方式

主要使用 Spring Boot 框架开发，Spring Boot 就可以理解为一个大型的包含的可能用的所有工具的一个工具箱，你可以快速的从里面拿出常见的你想用的功能。像接下来说的 mybatis、JPA、Spring MVC 等都包括在其中。

# 数据库查询 （数据访问层）

最初使用使用数据库查询的步骤都是连接数据库 -> 查询数据 -> 断开连接 这么一个反复重复的过程

  1. 性能优化：数据库连接的建立和关闭是相对昂贵的操作。如果每次数据库操作都需要重新建立连接，将会增加不必要的开销，降低系统的性能。通过持久化连接，可以减少连接的建立和关闭次数，从而提高数据库操作的效率。
  2. 资源利用：数据库连接是有限资源，数据库服务器同时能够处理的连接数是有限的。如果每次操作都建立新的连接，可能会导致连接池耗尽，无法为其他请求提供连接。通过持久化连接，可以更好地管理和利用连接资源。
  3. 减少延迟：建立数据库连接涉及网络通信和认证等步骤，会引入一定的延迟。如果频繁地建立和关闭连接，会增加请求的响应时间。通过持久化连接，可以减少这种额外的延迟，提升用户体验。
  4. 事务管理：在一个事务内执行多个数据库操作时，需要保持相同的数据库连接，以确保这些操作在同一个事务中。持久化连接可以保证事务内的多个操作使用同一个连接，从而维护事务的一致性和隔离性。

数据库查询是指从数据库中检索所需数据的过程。在系统开发中，可以使用不同的持久化框架来执行数据库查询，目前常用的主要有 MybatisPlus 和 JPA。

# MybatisPlus

mybatis 本质便是让系统自动生成一个 Sql 查询语句或者让你可以更方便更灵活的去编写一个复杂的查询语句。

# JPA

JPA 是一种规范，用于在 Java 应用程序和数据库之间实现对象关系映射（ORM）。

大概可以类比为将你数据库中的一个表和系统中的一段代码关联起来，让某一段代码代表整个表，操作那段代码就是操作对应的表。

# 业务处理（Service 层）

Service 层是软件开发中的一个重要组成部分，用于处理业务逻辑和业务操作。

它的主要目的是将控制器（或其他接口层）与数据访问层（如 DAO 层）解耦，同时负责封装业务逻辑的具体实现。

在典型的三层架构中，Service 层位于控制器层和数据访问层之间，起到协调和逻辑处理的作用。

特点：

  1. 业务逻辑的集中管理：Service 层作为业务逻辑的集中处理部分，将相关的业务操作组织在一起，提高了代码的可维护性和可重用性。
  2. 解耦合：Service 层将控制器（或其他接口层）与数据访问层隔离开来，使得不同层的代码更加独立，减少了各层之间的耦合性。
  3. 事务管理：Service 层常常涉及到事务的处理，确保一系列业务操作在同一事务中执行，保证数据的一致性和完整性。

主要功能：

  1. 业务逻辑处理：Service 层负责实现业务逻辑，如数据计算、验证、转换等操作。
  2. 数据处理：Service 层调用数据访问层（如 DAO 层）进行数据库操作，封装了数据的增删改查等操作。
  3. 错误处理与异常处理：Service 层可以捕获并处理各种可能发生的异常，向上层提供合适的错误信息。
  4. 事务管理：Service 层负责管理事务，确保多个数据库操作在同一个事务中执行，保证数据一致性。
  5. 业务流程管理：Service 层可以协调多个不同的业务操作，形成完整的业务流程。

# 页面控制器（Controller 层）

目前主要是使用 Spring MVC 来完成对各种访问，各种请求的接收和响应，其实 Spring MVC 主要的作用还是简化操作，可以将原本要上百行代码才能实现的功能简化到只需要几行，来极大的保证项目的下限。

特点：

  1. 处理用户请求：Controller 层接收并解析用户的请求，根据请求的类型和路径，决定执行哪些业务逻辑。
  2. 分发请求：根据不同的请求，将其分发给相应的业务逻辑处理方法。
  3. 协调业务逻辑：Controller 层协调不同的 Service 方法，确保业务逻辑按照正确的顺序执行。

主要功能：

  1. 请求映射：Controller 层使用注解来定义哪些 URL 路径对应哪些方法，将用户请求映射到特定的处理方法。
  2. 请求参数解析：Controller 层负责解析 URL 路径参数、查询参数、表单数据等，并将其传递给相应的业务逻辑方法。
  3. 调用 Service 方法：Controller 层调用 Service 层的方法来处理具体的业务逻辑，实现业务功能。
  4. 响应构建：根据业务处理的结果，Controller 层构建合适的响应对象，将数据返回给前端或客户端。

# 消息队列

消息队列（Message Queue）是一种在分布式系统中用于异步通信的技术。它允许不同的应用程序或服务在解耦的情况下进行通信，通过在消息队列中传递消息来实现数据传输。消息队列在各种场景下都有广泛的应用，包括异步任务处理、解耦服务、实现发布 - 订阅模式等。

特点：

  1. 解耦性：消息队列可以将发送者和接收者解耦，使它们不需要直接通信。发送者将消息发送到队列，接收者从队列中获取消息，从而降低了系统组件之间的耦合度。
  2. 异步通信：消息队列允许消息的发送和接收在不同的时间和速率下进行，从而实现异步通信。发送者发送消息后不需要立即等待响应，而是可以继续进行其他工作。
  3. 可靠性：许多消息队列系统提供持久化功能，确保即使在消息发送后发生系统崩溃等情况，消息也不会丢失。
  4. 扩展性：消息队列系统可以水平扩展，允许增加更多的消息代理和消费者以处理更大的工作负载。

主要用途：

  1. 任务队列：将耗时的任务放入消息队列，由消费者异步处理，提高系统的响应速度。
  2. 解耦服务：不同的服务通过消息队列进行通信，实现松耦合，使得系统更容易维护和扩展。
  3. 发布 - 订阅：实现一对多的消息传递模式，订阅者可以订阅感兴趣的主题，发布者发布消息给特定主题。
  4. 日志传输：将日志消息发送到消息队列，由消费者进行日志分析和处理。
  5. 事件驱动：通过消息队列传递事件，使不同的服务可以响应和处理事件。

# 流程管理

流程管理系统是一种用于管理和执行业务流程的软件系统。它可以帮助组织有效地设计、执行和监控各种业务流程，从而提高工作效率、减少错误和延迟，并支持更好的业务决策。这种系统通常用于自动化和优化复杂的业务流程，从而提供更好的可见性、控制和报告。

以下是流程管理系统的一些主要特点和用途：

特点：

  1. 流程建模：流程管理系统允许用户以图形化方式建模业务流程，定义流程中的步骤、活动和分支条件等。
  2. 自动化执行：流程管理系统可以自动执行流程，根据预定义的规则和逻辑，自动转移任务、通知参与者等。
  3. 任务分配：系统可以将任务分配给合适的人员或角色，确保流程按照规定的责任链执行。
  4. 流程监控和跟踪：系统可以提供实时的流程监控，让用户了解每个流程实例的状态、进度和历史记录。
  5. 报告和分析：流程管理系统可以生成报告和分析，帮助管理层了解业务流程的效率和问题。

主要用途：

  1. 审批流程：流程管理系统可以用于管理各种审批流程，如请假申请、采购审批等。
  2. 工作流程：适用于管理复杂的业务流程，如订单处理、客户服务请求处理等。
  3. 流程自动化：帮助自动化一些例行的任务，提高工作效率。
  4. 风险管理：通过规定的流程来管理风险，确保合规性和减少风险。
  5. 提升协作：通过自动任务分配和协作功能，增强团队的协作能力。

# 数据存储

# Cookie

Cookie 是一小段文本信息，由服务器发送给浏览器，浏览器将其存储在用户的计算机上。每次浏览器向服务器发送请求时，会将相关的 Cookie 信息附加在请求头中。

主要用于记录用户的状态信息，如登录状态、用户偏好设置、跟踪用户行为等。

特点：

  • 存储在客户端，有大小限制（通常为 4KB）。
  • 可以设置过期时间，在过期前一直存在。
  • 可以由服务器或客户端修改。
  • 不适合存储敏感信息，因为可以在客户端被访问和编辑。

# Session

Session 是一种服务器端的机制，用于存储和管理用户的状态信息。在服务器上创建一个会话，为每个会话分配唯一的标识（通常是 Session ID ），浏览器通过 Cookie 将 Session ID 保存，以在每次请求时进行会话标识。

主要用于管理用户的登录状态、购物车、临时数据等。

特点：

  • 存储在服务器端，相对安全，可以存储敏感信息。
  • 会话通常有时限，用户在一段时间内无操作会话失效。
  • 需要服务器支持，会占用服务器资源。

# LocalStorage

localStorage 是浏览器提供的一种本地存储机制，可以在浏览器中永久性地存储键值对数据，不受会话结束或浏览器关闭影响。

主要用于在客户端存储数据，例如保存用户的设置、缓存数据等。

特点：

  • 存储在客户端，较大的存储容量（通常为 5MB）。
  • 永久性存储，除非用户主动删除或清除浏览器数据。
  • 仅存储为字符串，需要手动转换为其他数据类型。
  • 适合存储非敏感数据，如用户设置、缓存等。

# 界面控制

# HTML (HyperText Markup Language)

HTML 是一种标记语言，用于描述网页的结构和内容。它通过使用各种标签和元素来定义文本、图像、链接以及其他内容的组织方式。

主要用于创建网页的结构，定义页面中的标题、段落、列表、表格、超链接等元素，使内容有逻辑结构。

创建网页标题 <h1> 、段落 <p> 、链接 <a> 、图像 <img> 等。

# CSS (Cascading Style Sheets)

CSS 是一种样式表语言，用于定义网页的外观和样式。它通过选择器和属性来控制 HTML 元素的外观，如颜色、字体、布局等。

主要用于美化和布局网页，实现页面的视觉效果，使页面更加吸引人和易于阅读。

定义文本颜色 color 、背景颜色 background-color 、字体大小 font-size 等。

# JavaScript

JavaScript 是一种脚本语言，用于在网页上添加交互性和动态行为。它可以与用户交互、操作 DOM（文档对象模型）、发送网络请求等。

主要用于实现网页的交互功能，如表单验证、动画效果、响应用户事件等，以及与服务器通信。

响应按钮点击事件、验证用户输入、动态更新页面内容等。
